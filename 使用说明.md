# Git 提交影响评估：傻瓜式启动说明

目的：分析一个 Git 仓库最近 `NUM` 次提交，并生成报告。

## macOS 用户

### 第 1 步：进入项目目录

把 `/你的项目目录/AgentGitCheck` 换成项目实际位置：

```bash
cd /你的项目目录/AgentGitCheck
```

### 第 2 步：安装依赖

第一次运行执行：

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
```

以后再次运行，只需要执行：

```bash
cd /你的项目目录/AgentGitCheck
source .venv/bin/activate
```

### 第 3 步：创建配置文件

```bash
cp .env.example .env
open -e .env
```

在 `.env` 中修改这 3 项：

```env
REPO_PATH=/Users/你的用户名/要分析的仓库
NUM=30
OUTPUT_PATH=git-agent-report.md
```

例如：

```env
REPO_PATH=/Users/zhangsan/projects/my-repository
NUM=100
OUTPUT_PATH=git-agent-report.md
```

`NUM=100` 表示分析最近 100 次提交。

模型请求失败默认会重试 3 次；可在 `.env` 中调整：

```env
MAX_RETRIES=3
RETRY_DELAY_SECONDS=2
```

第一次测试可以把 `API_KEY` 留空，先运行离线基础模式。

### 可选：飞书机器人通知

需要在分析完成后将摘要发送到飞书时，在 `.env` 中填写：

```env
FEISHU_ENABLED=true
FEISHU_WEBHOOK_URL=你的飞书机器人Webhook地址
```

不需要通知时保持 `FEISHU_ENABLED=false`。

飞书 Webhook 机器人会发送报告第 2、3、4 节的压缩摘要卡片。

### 第 4 步：启动

```bash
python run.py
```

看到下面内容就表示成功：

```text
分析完成：.../git-agent-report.md
提交数：30，总体风险：low
```

打开报告：

```bash
open git-agent-report.md
```

## Windows 用户

### 第 1 步：进入项目目录

```powershell
Set-Location E:\AgentGitCheck
```

### 第 2 步：安装依赖

第一次运行执行：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
```

以后再次运行，只需要执行：

```powershell
Set-Location E:\AgentGitCheck
.\.venv\Scripts\Activate.ps1
```

### 第 3 步：创建配置文件

```powershell
Copy-Item .env.example .env
notepad .env
```

在 `.env` 中修改这 3 项：

```env
REPO_PATH=E:\你的目标仓库路径
NUM=30
OUTPUT_PATH=git-agent-report.md
```

例如：

```env
REPO_PATH=E:\你的项目目录\your-git-repository
NUM=100
OUTPUT_PATH=git-agent-report.md
```

第一次测试可以把 `API_KEY` 留空，先运行离线基础模式。

### 第 4 步：启动

```powershell
python run.py
```

打开报告：

```powershell
notepad .\git-agent-report.md
```

## 只想临时改分析次数

不改 `.env`，直接运行：

macOS：

```bash
python run.py --limit 10
```

Windows：

```powershell
python run.py --limit 10
```

这次会分析最近 10 次提交。

## 遇到 Git 安全错误时

如果看到 `detected dubious ownership`，先执行下面命令，再启动程序。

macOS：

```bash
export GIT_CONFIG_COUNT=1
export GIT_CONFIG_KEY_0=safe.directory
export GIT_CONFIG_VALUE_0="/Users/你的用户名/要分析的仓库"
python run.py
```

Windows PowerShell：

```powershell
$env:GIT_CONFIG_COUNT = "1"
$env:GIT_CONFIG_KEY_0 = "safe.directory"
$env:GIT_CONFIG_VALUE_0 = "E:\你的目标仓库路径"
python run.py
```

## 成功标准

满足以下条件即可：

1. 命令没有报错。
2. 出现 `分析完成`。
3. 项目目录生成 `git-agent-report.md`。
4. 报告中的提交数不超过 `NUM`。
